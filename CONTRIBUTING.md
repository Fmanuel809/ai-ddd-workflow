# Contributing

Thanks for your interest in contributing to AI DDD Workflow.

Maintainer: [Felix M. Martinez](https://github.com/fmanuel809)

## Getting Started

1. Fork the repository.
2. Create a branch from the default branch (`master`) with a clear name, for example: `feat/stage-a4-updates`.
3. Make focused changes with clear commit messages.
4. Open a pull request using the provided PR template.

## Development Guidelines

- Keep changes aligned with `AGENTS.md` and `ddd-config.yml` policies.
- Prefer small, reviewable PRs with one concern per PR.
- Update related docs and artifacts when behavior or governance changes.
- Keep artifact and stage naming consistent with the A0-A7 model.

## Quality Expectations

- Ensure the repository remains internally consistent.
- Validate links and paths in documentation.
- Include rationale for architectural or governance-impacting changes.

## Pull Request Checklist

- [ ] Scope is clear and limited.
- [ ] Documentation is updated.
- [ ] Changes follow repository conventions.
- [ ] No sensitive information is included.

## Code of Conduct

By participating, you agree to follow the [Code of Conduct](CODE_OF_CONDUCT.md).
